Prinsip Single Responsibility Principle atau SRP sangat membantu saya dalam proses pengembangan aplikasi ini, khususnya saat menambahkan fitur History Logger pada counter. Dengan menerapkan SRP, setiap bagian dari program memiliki tanggung jawab yang jelas dan tidak saling bercampur.
CounterController hanya berfokus pada pengolahan logika aplikasi, seperti menambah dan mengurangi nilai counter, mengatur nilai step, melakukan reset, serta menyimpan data riwayat perubahan counter. Sementara itu, CounterView hanya bertugas untuk menampilkan data ke layar dan menangani interaksi dari pengguna, seperti menekan tombol atau mengisi TextField.
Ketika saya menambahkan fitur riwayat, saya tidak perlu mengubah banyak kode pada bagian tampilan. Saya hanya menambahkan logika pencatatan riwayat di Controller, lalu menampilkannya di View. Hal ini membuat proses pengembangan menjadi lebih terstruktur, lebih mudah dipahami, dan mengurangi kemungkinan terjadinya kesalahan. Dengan menerapkan prinsip SRP, kode menjadi lebih rapi, mudah dikembangkan di kemudian hari, serta lebih mudah untuk diperbaiki jika terjadi error.
